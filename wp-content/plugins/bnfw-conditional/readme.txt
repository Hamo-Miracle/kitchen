Allows you to restrict notification to be sent only when specific conditions are met.

== Changelog ==

= 1.2 =
* New: AND/OR logic for post/page/custom post type notifications. You can choose to only send a notification if All or Any categories/tags/taxonomy terms match for a given notification.

= 1.1.2 =
* Fixed: An issue where enabling/disabling a conditional notification wasn't taking effect.

= 1.1.1.2 =
* Version bump. Revised 'Tested up to' and added 'Requires Plugins' header.

= 1.1.1.1 =
Fixed: An issue where conditional form elements weren't showing when creating/editing a notification.

= 1.1.1 =
* Added: Support for Conditional Notifications in Multisite add-on.

= 1.1 =
* New: Added support for plugins that allow multiple user roles to be assigned to a user.

= 1.0.16 =
* Fixed: Conditional fields were showing in notifications that didn't support them.

= 1.0.15 =
* Fixed: Conditional fields shouldn't be shown when a taxonomy is selected.

= 1.0.14 =
* Fixed: Conditional fields weren't showing correctly for the new comment notifications.
* Removed: Conditional fields from the Media notifications as they're not currently used.

= 1.0.13 =
* Improved: Support for new user notifications in the Multisite add-on.

= 1.0.12 =
* Fixed: This add-on now works when users are assigned to multiple user roles.

= 1.0.11 =
* Fixed: Conditional fields weren't showing for the New Comment and New Comment Awaiting Moderation notifications.

= 1.0.10 =
* Fixed: An issue where notifications weren't sending when used in conjunction with the Reminders add-on.

= 1.0.9 =
* Added: Support for conditional fields in the the new Comment Moderation notifications.

= 1.0.8 =
* Added: Inline help tips are now available for this add-ons notification fields.

= 1.0.7 =
* Added: Empty categories / terms / tags are now shown in the select box when setting conditional options on your notifications.

= 1.0.6 =
* Fixed: 'User Role Changed' notifications were being sent when a user was moved from and to any role.

= 1.0.5 =
* New: You can now limit 'New User Registration - For User' notifications to users who register for a specific user role.

= 1.0.4 =
* New: You can now send out notifications when a user transitions from one role to another when using the 'User Role Changed - For Admin' and 'User Role Changed - For User' notifications.

= 1.0.3 =
* New: Translation file now provided.
* Fixed: You couldn't previously remove any conditional settings from a notification. Now, when they're removed, a notification will work as if no conditional settings are present.

= 1.0.2 =
* Fix for conditional select fields showing on the 'New User Registration - For User' notification.

= 1.0.1 =
* Fix for conditional select fields not showing for custom post types that contain a hyphen in their name.

= 1.0 =
* Initial release.
